# Trace A：产品与 PRD 审计

## 初始结论

初始实现中，FR-02/03/04/05/08/10 基本完成；FR-01/06/07/09/11/12/13 只有部分证据。P1 的数据质检、报告版本和批次对比后端较完整，导出与操作审计存在边缘缺口。

## 发现的关键问题

- `client_request_id` 仅按 ID 重放，没有校验请求内容。
- 后端未强制 `batch_name`，采样 UI 缺少 tags。
- 批次详情未展示不可变配置快照，证据不能定位具体 run。
- analytics summary 缺少报告版本、配置口径和数据截止时间。
- source retry 缺少提交前精确数量。
- 兼容 `/rerun_failed` 路径未写审计。
- 冻结报告没有前端导出入口。

## 修复后复核方向

本轮已补请求指纹冲突、后端名称校验、tags、配置快照、证据跳转、summary 元数据、source 数量、兼容审计与冻结快照导出。最终签署仍需以 HTTP/E2E 实测为准。
